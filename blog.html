What is event delegation in the context of the DOM,and why is it useful?

Event delegation is a technique in JavaScript where you attach a single event listener to a parent element to manage events for multiple child elements, rather than attaching event listeners to each individual child. This is especially useful in scenarios where child elements are dynamically added or removed from the DOM.

Event delegation is useful because it improves performance by reducing the number of event listeners needed in the DOM. Instead of attaching a listener to each child element, you attach one to a common parent. This also simplifies handling dynamically added elements, as the single parent listener will automatically capture events from new children, making the code more efficient and easier to maintain.

How do you manipulate an element's attributes and styles using the DOM?

You can manipulate an element's attributes and styles using the DOM by leveraging built-in methods and properties in JavaScript. To modify attributes, you can use methods like setAttribute() to add or change an attribute, getAttribute() to retrieve its value, and removeAttribute() to remove it. For styling, the style property allows direct manipulation of inline CSS, such as setting element.style.backgroundColor or other styles dynamically. You can also control CSS classes using classList methods, such as add(), remove(), and toggle(), making it easier to apply or modify styles through CSS classes.
